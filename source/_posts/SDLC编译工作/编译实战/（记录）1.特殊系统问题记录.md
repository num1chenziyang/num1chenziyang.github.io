# （一）gls系统报错
## 一、glu报错
```bash
### 报错信息：
ABORTING TEST - init glu call fails with error 8       

### 报错原因：
sh -c "ob/tgls 2> ob/tgls.log" 
### 打开gls的log文件，找到此命令复现报错，原因是找不到glu库。  

### 解决方法：
1.安装glu库。
2.让PATH可以找到glu库。对于sw而言，我使用软链接：
ln -s /usr/lib/sw_64-linux-gnu/libglusterfs.so.0.0.1 /usr/lib/libglusterfs.so.0
```
