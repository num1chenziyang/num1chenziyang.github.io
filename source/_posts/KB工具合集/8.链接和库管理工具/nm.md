nm 命令用于显示目标文件（如 .o、.a、.so 文件）或可执行文件中的符号表。输出的每一行通常包含一个符号的地址、类型和名称。其中，类型由一个字母表示，不同的字母代表了符号在程序中的不同性质和位置。理解这些字母对于调试和分析程序非常有用。  
以下是 nm 输出中常见的符号类型及其含义：  
**常见符号类型说明：**

|   |   |   |
|---|---|---|
|**符号**|**大小写**|**含义**|
|U|大写|**Undefined****（未定义）符号**。该符号在当前目标文件中未定义，但在链接时需要从其他目标文件或库中解析。|
|T|大写|**全局文本段符号**。符号在文本段（代码段）中定义，且是全局可见的。|
|t|小写|**局部文本段符号**。符号在文本段中定义，且仅在本地可见（静态函数等）。|
|D|大写|**全局已初始化数据段符号**。符号在已初始化的数据段中定义，且是全局可见的。|
|d|小写|**局部已初始化数据段符号**。符号在已初始化的数据段中定义，且仅在本地可见。|
|B|大写|**全局未初始化数据段符号**（BSS段）。符号在未初始化的数据段中定义，且是全局可见的。|
|b|小写|**局部未初始化数据段符号**。符号在未初始化的数据段中定义，且仅在本地可见。|
|R|大写|**全局只读数据段符号**。|
|r|小写|**局部只读数据段符号**。|
|G|大写|**全局小数据段符号**。用于一些架构上的小型全局变量（例如在Small Data区域）。|
|g|小写|**局部小数据段符号**。|
|S|大写|**全局小型未初始化数据段符号**（小型BSS段）。|
|s|小写|**局部小型未初始化数据段符号**。|
|C|大写|**Common****（公共）符号**。未初始化的全局变量，可能被多个目标文件定义。|
|V|大写|**全局弱符号**。与 W 类似，但针对对象而非函数。|
|v|小写|**局部弱符号**。|
|W|大写|**全局弱符号**。弱符号允许多个定义，链接器会选用一个。|
|w|小写|**局部弱符号**。|
|N|大写|**调试符号**。通常在编译时加入调试信息，如行号、文件名等。|
|?||**未知类型符号**。|

**详细解释：**

- **大小写区分：**
    
    - **大写字母**表示全局符号，链接器在其他目标文件中也可以访问。
    - **小写字母**表示局部符号，仅在定义它的目标文件内可见。
- **符号类型：**
    
    - **U****（****Undefined****）未定义符号：**
        
        - 该符号在当前目标文件中未定义，需要从其他目标文件或库中解析。
        - 例如，调用了外部库函数但未链接对应库时，会显示为未定义符号。
    - **T/t****（****Text section****）文本段符号：**
        
        - 存储程序的代码部分，即函数的实现。
        - 全局符号通常是可供外部调用的函数，局部符号是 static 修饰的函数。
    - **D/d****（****Data section****）已初始化数据段符号：**
        
        - 存储已初始化的全局或静态变量。
        - 例如，int a = 10;。
    - **B/b****（****BSS section****）未初始化数据段符号：**
        
        - 存储未初始化的全局或静态变量，程序开始时被初始化为零。
        - 例如，int a; 未被显式初始化。
    - **R/r****（****Read-only data section****）只读数据段符号：**
        
        - 存储只读数据，如常量字符串、const 修饰的全局变量等。
    - **G/g** **和** **S/s****（****Small data/BSS section****）小数据段符号：**
        
        - 用于一些处理器架构（如ARM、MIPS）上的小型数据优化。
        - 小数据段允许更高效地访问小型全局变量。
    - **C****（****Common symbol****）公共符号：**
        
        - 未初始化的全局变量，如果在多个文件中定义了同名的 common 符号，链接器会将它们合并。
        - 可以通过编译器选项或代码修改将其转为 BSS 符号。
    - **W/w** **和** **V/v****（****Weak symbol****）弱符号：**
        
        - 弱符号允许多个定义，链接器在解析时如果找到一个强符号（非弱符号），则使用强符号，否则使用弱符号。
        - 常用于库函数，可以被用户自定义的函数覆盖。
    - **N****（****Debug symbol****）调试符号：**
        
        - 包含调试信息，不参与程序运行。
        - 只有在编译时使用 -g 选项生成调试信息时才会出现。
